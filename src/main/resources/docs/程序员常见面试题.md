# 程序员常见面试题

## 一、Java基础面试题

### 1.1 面向对象
**Q1: 什么是面向对象？面向对象的三大特性是什么？**
- 面向对象是一种编程范式，将数据和操作数据的方法组织在一起
- 三大特性：封装、继承、多态

**Q2: 接口和抽象类有什么区别？**
| 特性 | 接口 | 抽象类 |
|------|------|--------|
| 方法实现 | JDK8前只能有抽象方法 | 可以有抽象和具体方法 |
| 多继承 | 支持多实现 | 只能单继承 |
| 构造方法 | 不能有 | 可以有 |
| 访问修饰符 | 默认public | 任意 |

**Q3: 重载(Overload)和重写(Override)的区别？**
- 重载：同一类中，方法名相同，参数列表不同
- 重写：子类对父类方法的重新实现，方法签名相同

### 1.2 集合框架
**Q4: ArrayList和LinkedList的区别？**
- ArrayList：基于数组，查询快O(1)，增删慢O(n)
- LinkedList：基于链表，查询慢O(n)，增删快O(1)

**Q5: HashMap的底层原理？**
- JDK7：数组+链表
- JDK8：数组+链表+红黑树（链表长度>8时转换）
- 默认初始容量16，负载因子0.75

**Q6: ConcurrentHashMap如何保证线程安全？**
- JDK7：分段锁（Segment）
- JDK8：CAS+synchronized

### 1.3 多线程
**Q7: 线程的几种状态？**
- NEW、RUNNABLE、BLOCKED、WAITING、TIMED_WAITING、TERMINATED

**Q8: synchronized和Lock的区别？**
- synchronized是关键字，Lock是接口
- Lock可以尝试获取锁、可中断、可超时
- Lock需要手动释放锁

**Q9: volatile关键字的作用？**
- 保证可见性：一个线程修改，其他线程立即可见
- 禁止指令重排序
- 不保证原子性

**Q10: 线程池的核心参数有哪些？**
- corePoolSize：核心线程数
- maximumPoolSize：最大线程数
- keepAliveTime：非核心线程存活时间
- workQueue：任务队列
- threadFactory：线程工厂
- handler：拒绝策略

## 二、JVM面试题

### 2.1 JVM内存模型
**Q11: JVM内存结构是怎样的？**
- 堆（Heap）：存放对象实例
- 方法区（Method Area）：类信息、常量、静态变量
- 虚拟机栈（VM Stack）：方法执行的内存模型
- 本地方法栈（Native Method Stack）：Native方法服务
- 程序计数器（PC Register）：当前线程执行位置

**Q12: 垃圾回收算法有哪些？**
- 标记-清除算法
- 复制算法
- 标记-整理算法
- 分代收集算法

**Q13: 常见的垃圾收集器有哪些？**
- Serial/Serial Old：单线程，适合小内存
- ParNew：Serial的多线程版本
- Parallel Scavenge/Parallel Old：吞吐量优先
- CMS：低停顿
- G1：区域化分代收集
- ZGC/Shenandoah：低延迟

**Q14: 什么是类加载机制？**
- 加载 → 验证 → 准备 → 解析 → 初始化
- 双亲委派模型：Bootstrap → Extension → Application → Custom

## 三、Spring面试题

### 3.1 Spring基础
**Q15: 什么是IoC和AOP？**
- IoC（控制反转）：将对象创建和管理交给Spring容器
- AOP（面向切面编程）：将横切关注点与业务逻辑分离

**Q16: Spring Bean的生命周期？**
1. 实例化
2. 属性赋值
3. 初始化（InitializingBean、@PostConstruct）
4. 使用
5. 销毁（DisposableBean、@PreDestroy）

**Q17: Spring事务的传播行为有哪些？**
- REQUIRED（默认）：当前无事务则新建，有则加入
- REQUIRES_NEW：新建事务，挂起当前事务
- NESTED：嵌套事务
- SUPPORTS：有事务则加入，无则以非事务执行
- MANDATORY：必须有事务，否则抛异常

### 3.2 Spring Boot
**Q18: Spring Boot的自动配置原理？**
- @EnableAutoConfiguration注解
- META-INF/spring.factories文件
- @Conditional条件注解

**Q19: Spring Boot Starter是什么？**
- 预定义的依赖集合，简化Maven配置
- 自动配置相关组件

## 四、数据库面试题

### 4.1 MySQL
**Q20: MyISAM和InnoDB的区别？**
| 特性 | MyISAM | InnoDB |
|------|--------|--------|
| 事务 | 不支持 | 支持 |
| 锁 | 表锁 | 行锁+表锁 |
| 外键 | 不支持 | 支持 |
| 崩溃恢复 | 不支持 | 支持 |

**Q21: 索引的类型有哪些？**
- B+树索引（聚簇索引、非聚簇索引）
- 哈希索引
- 全文索引
- 空间索引

**Q22: 什么是SQL优化？**
- 避免SELECT *
- 使用索引
- 避免在索引列上使用函数
- 优化JOIN操作
- 使用EXPLAIN分析执行计划

**Q23: 事务的ACID特性？**
- Atomicity（原子性）
- Consistency（一致性）
- Isolation（隔离性）
- Durability（持久性）

**Q24: 事务隔离级别有哪些？**
- READ UNCOMMITTED
- READ COMMITTED
- REPEATABLE READ（MySQL默认）
- SERIALIZABLE

### 4.2 Redis
**Q25: Redis的数据类型及应用？**
- String：缓存、计数器
- Hash：对象存储
- List：消息队列
- Set：去重、交集并集
- ZSet：排行榜

**Q26: Redis持久化方式？**
- RDB：快照，定期保存
- AOF：日志，记录写操作
- 混合模式（Redis 4.0+）

**Q27: 缓存穿透、击穿、雪崩及解决方案？**
- 穿透：查询不存在数据 → 布隆过滤器
- 击穿：热点key过期 → 互斥锁、永不过期
- 雪崩：大量key同时过期 → 随机过期时间、多级缓存

## 五、微服务面试题

**Q28: 微服务的优缺点？**
优点：
- 独立部署
- 技术栈灵活
- 扩展性强
- 团队自治

缺点：
- 分布式复杂性
- 运维成本高
- 数据一致性挑战

**Q29: 服务注册与发现的原理？**
- 服务启动时向注册中心注册
- 客户端从注册中心获取服务列表
- 心跳机制保活

**Q30: 什么是服务熔断和降级？**
- 熔断：故障时快速失败，防止雪崩
- 降级：非核心功能关闭，保证核心功能

## 七、算法与数据结构面试题

**Q34: 常用的排序算法及时间复杂度？**
| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|----------|----------|------|--------|
| 冒泡 | O(n²) | O(n²) | O(1) | 稳定 |
| 快排 | O(nlogn) | O(n²) | O(logn) | 不稳定 |
| 归并 | O(nlogn) | O(nlogn) | O(n) | 稳定 |
| 堆排 | O(nlogn) | O(nlogn) | O(1) | 不稳定 |

**Q35: 二叉树的前序、中序、后序遍历？**
- 前序：根-左-右
- 中序：左-根-右
- 后序：左-右-根

**Q36: 什么是红黑树？**
- 自平衡二叉查找树
- 节点是红色或黑色
- 根节点是黑色
- 红色节点的子节点是黑色
- 从任一节点到叶子的黑节点数相同
